The Warden CLI reviews local changes, manages skills, and helps configure
GitHub integration.

```bash
warden [targets...] [options]
```

If no command is given, Warden runs the default analysis command. With no
target, it reviews the current branch against the default branch.

## Global Options

These options are available at the root CLI level. Command-specific options are
documented on each command page.

| Option | Purpose |
| --- | --- |
| `-C, --cwd <path>` | Run as if invoked from this directory. |
| `-h, --help` | Show help for a command. |
| `-V, --version` | Show version number. |

## Common Examples

```bash
warden                     # Review current branch changes
warden src/auth.ts         # Analyze a specific file
warden src/api/            # Analyze a directory
warden @targets.txt        # Analyze targets listed in a file
warden --staged            # Analyze staged changes
warden HEAD~3..HEAD        # Analyze changes in a git range
warden help add            # Show command help
```
